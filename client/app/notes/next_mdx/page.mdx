# Next.js の @next/mdx を導入してみた

このサイト、技術の実験場的な意味合いを込めて labs というサブドメインにしているんですが、だったら技術メモを書きたいなとなり Next.js でサクッと書いて公開する方法がないか検討してみたところ、公式の@next/mdx がよさそうだったので試してみました。

[公式のドキュメント](https://nextjs.org/docs/app/building-your-application/configuring/mdx)

これだけだとうまくいかなかったので[Zenn の記事](https://zenn.dev/hayato94087/articles/1411f7af3eaee4)なんかも参考にしながら、`/notes`というパスで無事技術メモを公開することができました。

[GitHub リポジトリ](https://github.com/horiyee/horiy.dev)の`/client`下にコードも push しています。

導入方法は上のリンクと実際のコードを見ればわかると思うので割愛しますが、入れてすぐにこの記事を書いていて[Pico CSS](https://picocss.com/)との相性もめっちゃよくてフロントエンドをいいかんじに頑張りすぎずに作れたな、と思っています。

ただ、Next.js をちゃんと触るのが実に 2, 3 年ぶりというのもあって、これまで逃げ続けてきた App Router と向き合わなければいけなかったり、[Pico CSS のパンくずリスト](https://picocss.com/docs/nav#breadcrumb)を使いたいのに layout.tsx 何もわからないになっていてまだいいかんじにできていなかったりと改善の余地がめちゃくちゃありそうなのでいろいろ試していきたい所存。
